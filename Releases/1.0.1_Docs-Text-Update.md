## **Updated the Text component**
-------------------------------------------------------
The `Text` primitive now renders semantic HTML by default and resets margins unconditionally.

**What changed:**
- Added `as` prop to override the rendered element (`h1`, `h2`, ..., `p`, `span`, `label`, `div`)
- Heading sizes (`h1`–`h6`) now render as their corresponding heading elements (previously all rendered as `<p>`)
- Body sizes (`body1`, `body2`) now render as `<span>` (neutral, inline, no margin)
- All elements now have `m-0` applied, so spacing is controlled by `Stack`/`Inline` gaps — not browser defaults

**Why:**
The Velella team reported that `Text` was injecting unexpected spacing via browser-default paragraph margins. This made layout debugging confusing because the gap appeared to come from Counterfoil primitives when it actually came from the user-agent stylesheet. The fix ensures `Text` fully owns its layout behavior.

**Migration:**
- Existing call sites get better defaults automatically (semantic headings, no margins)
- If you relied on paragraph margins for spacing, use `Stack` gap or explicit margin utilities instead
- If you need a specific element, use the new `as` prop: `<Text size="body1" as="p">...</Text>`
